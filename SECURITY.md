# Security Policy

## Supported Versions

We currently support the latest version on the `main` branch. Older releases no longer receive security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not open a public GitHub issue**. Instead, email **x18825407105@outlook.com** with the following information:

- Affected component or file path
- Steps to reproduce (proof of concept)
- Impact assessment and affected versions
- Your suggested fix (optional)

We will acknowledge receipt within **48 hours** and aim to provide an initial assessment within **5 business days**. Please give us a reasonable disclosure window (default **90 days**) before publishing details publicly, so we can ship a fix and protect users.

## Scope

This policy covers the code in this repository. Vulnerabilities in third-party dependencies should be reported upstream. Out-of-scope findings include: self-XSS, missing security headers already addressed by the Content Security Policy, and issues that require physical access to a victim's device.
